the setup

  • in ac306, there are 4 laptops set up on 2 networks
  • left = hub network, right = switched network
  • there are active network process running on the console of all laptops, creating traffic from one node to another on the same network

do (network)

  • discover the machine IP addresses
  • discover the (2) username and passwords on each machine
  • gain access to the machines using network sniffing only.
  • prove your exploits by compromising the web server and user accounts and recording your username in the obvious places
  • write it up, mostly with screen shots and some brief explanations as a PDF and email it to me by 11/2

do (forensics)

  • kevin rooted the wargames server, find out how by inspecting the logs that I am emailing you
  • I am including all the logs, only some of which are useful. There is enough information to figure out what exploit kevin used. You can also poke around the wargames server looking on your own.
  • send me email on how kevin rooted the machine, with a pointer to the exploit online

notes

  • (hub) you should only need to sniff using wireshark, and look up some stuff on HTTP Basic Authentication hashes
  • (switched) you need to use ARP poisoning and IP forwarding to perform a man-in-the-middle attack
  • the consoles are for status information only, do not touch
  • google search is your friend when looking for kevin's exploit vector
